部署服务器

January 12, 2022 · 3 min read

榨干服务器，这当然是一句玩笑话，榨干性能还不是简简单单吗，下面罗列几个简单的应用

1.nginx转发，不暴露其他端口​

    proxy_pass  http://127.0.0.1:8088; # 转发规则
    proxy_set_header Host $proxy_host; # 修改转发请求头，让8080端口的应用可以受到真实的请求
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

在各个路径加入转发点，连带请求头一起转发

2.安装netdata​

netdata是一个优秀的在线服务器信息查看面板。


bash <(curl -Ss  https://my-netdata.io/kickstart-static64.sh)
systemctl start netdata

3.部署可道云​

wget https://static.kodcloud.com/update/download/kodbox.1.25.zip
unzip kodbox.1.25.zip && chmod -Rf 777 ./*

可道云是一个网盘系统。支持在线桌面，支持在线文档的优秀网盘系统。部署环境 lnmp

4.部署为知笔记​

访问阿里云控制台获取镜像加速链接

运行环境docker，配置docker镜像源

sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://db5sqf0y.mirror.aliyuncs.com"]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker

安装命令


docker run --name wiz --restart=always -it -d -v  ~/wizdata:/wiz/storage -v 
 /etc/localtime:/etc/localtime -p 8081:80 -p 9269:9269/udp  wiznote/wizserver

5.部署oneManger​

一个在线onedriver操控面板服务。需要申请onedriverAPI

当前已经部署的服务​

服务器就是要搭积木，一个一个服务进行搭建，需要什么就写什么。

全部使用nginx转发的好处就是，能够使用二级域名定向到端口，开启https服务，增强安全性，同时可以配置nginx防火墙，进一步保护服务器

坏处就是nginx崩溃的话，会影响到全部的服务
